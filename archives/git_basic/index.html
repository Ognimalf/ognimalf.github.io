Git Basic

October 30, 2021

git分支#

git branch: 用来创建分支.

git branch -f <n1> <n2>: 将分支n1强制移动到n2提交处.

<branch>^: 指向当前提交父节点,在^加数字可以选择返回的父节点.

<branch>~(num): 指向当前提交父节点路径,num表示回退提交数目

PS:符号^与~可以结合在一块实现更高效的回退父提交

git rebase <n>: 将当前的分支提交复制到指定分支上.

git rebase <m> <n>: 将分支n移动到分支m上

例如:

before:
            c1
            /\
           /  \
         c2    c4
         |     |
         |     |
         c3    c5
               |
               |
               c6*(HEAD)

git rebase c2
[]表示提交消失
               c1
          ____/  \____
          |           |
         c2          [c4]
         / \          |
        /   \         |
       c3    c4      [c5]
             |        |
             |        |
             c5       [c6]*(HEAD)
             |
             |
             c6*(HEAD)

git撤销提交#

git reset用于撤销本地提交.

git revert用于远程分支.该命令产生一个新的提交,用来撤销之前的提交.revert之后就能将更改推送到远程仓库.

git整理提交记录#

git cherry-pick <a1> ...: 将一些提交复制到当前分支下.

git rebase -i <b>: 交互式rebase,能够打开交互页面,包含当前提交b之后的内容,手动进行修改.

git Tag#

git的tag可以被永久固定在一个commit上.不会随提交移动而改变.使用者也不能切换到某个标签上进行修改.
git tag <tag_name> <commit>: 为提交commit设置标签名,默认为HEAD所在提交设立标签.

git describe <ref>

<ref> 可以是任何能被 Git 识别成提交记录的引用，如果你没有指定的话，Git 会以你目前所检出的位置(HEAD).

它输出的结果是这样的：

<tag>_<numCommits>_g<hash>

tag 表示的是离 ref 最近的标签， numCommits 是表示这个 ref 与 tag 相差有多少个提交记录， hash 表示的是你所给定的 ref 所表示的提交记录哈希值的前几位。

当 ref 提交记录上有某个标签时，则只输出标签名称.

git远程分支#

git远程分支的命名规范如下:

  • <remote name>/<branch name>

remote name就是远程仓库的名称.
origin作为远程仓库的默认名字.

git clone:在本地创建一个远程仓库的拷贝,对应的远程仓库名字默认为origin

git fetch: 作为本地与远程仓库的通信方式.这条命令将本地仓库中的远程分支更新成远程仓库对应分支的最新状态,但是并没有修改本地的文件,可以被看作单纯的下载操作.

git pull: 效果等同于git fetch + git merge.

git push: 负责将本地的变更上传到指定的远程仓库,并在远程仓库上合并新的提交记录,同时本地的远程分支也将被更新.注意,git push不带参数时行为与push.default的配置有关.

远程提交时,需要先保持本地分支与远程分支的一致性.可以使用

  • git rebase
  • git merge <b>: 结合分支b与当前的HEAD提交作为父节点,生成新的提交
  • git pull --rebase:将本地工作rebase到远程分支的最新提交记录

等命令实现目的.

注意:本地合并远程分支时,不要忘记使用git fetch从远端复制记录到本地分支

rebase与merge的比较#

rebase的优点:
Rebase 使你的提交树变得很干净, 所有的提交都在一条线上
rebase的缺点:
Rebase 修改了提交树的历史

远程跟踪#

设置本地跟踪分支:

  1. git checkout -b <local> <remote>:设置本地分支local跟踪远程分支remote,这意味着可以在local上使用git push,工作将被推送到remote分支上.
  2. git branch -u <remote> <local>:效果同上,local参数不给出时默认为当前所在分支.

注意:branch -u指定的local分支需要首先存在,不会创建分支.checkout -b会创建不存在的本地跟踪分支并自动指向它.

参数#

push#

git push <remote> <place>: place指定远程仓库remote需要查找的分支,push首先切换到本地的place指向的分支,获取所有的提交,然后在远程仓库(remote)中找到对应的分支,添加远程分支中没有的提交.

同样,这里的<place>参数可以被替换为<source>:<destination>,因此命令变成了如下形式:

git push origin <source>:<destination>

这里的参数意思是将本地source分支与destination远程分支绑定,确定在本地和远程需要定位的分支.配合之前分支的^与~定位符,push命令可以被更灵活的执行.如果destination分支不存在则会被创建.

fetch#

git fetch也有类似于push的参数,只是fetch的源位置在远程仓库,目标位置在本地分支,同样不存在的目标分支将会被建立.

需要注意的是,fetch只是作为下载记录的命令,因此将远程的提交放在本地的远程分支中.

fetch命令不提供参数的形式会将所有的提交下载至本地.

不指定source#

对于push命令,不指定source时,会将远程仓库中的destination分支删除;而对于fetch指令,则会创建一个destination的本地分支.

pull#

pull命令本质上作为fetch与merge的组合技能.例如:
git pull origin foo = git fetch origin foo; git merge o/foo git pull origin bar~1:bugFix = git fetch origin bar~1:bugFix; git merge bugFix

git指令速查#

作为初学者,在这里列举一些常用的git命令,以便随时查阅

  1. git init 初始化当前文件夹,创建本地仓库
  2. git config user.name , git config user.email 查看git账号的用户名与email地址
  3. git config --global user.name xxxxx , git config --global user.email xxxxx@xx.xxx 修改git账号的用户名与email
  4. git clone https/ssh 从github上克隆仓库
  5. git add 可以将文件添加到暂存区, git add . 将所有文件添加到暂存区, 添加后可以用git status -s 查看文件
  6. git rm --cache FILE 删除暂存区文件
  7. git commit 提交最新的代码, 可用选项 -a 能够提交修改过的文件并提交
  8. git log 可以查看commit的历史
  9. git rebase 在commit之后,最新的提交并不与远程的分支相同,此时需要合并commit
  10. git pull 在git commit之后如果直接push操作可能导致失败,因为本地仓库落后于远程仓库,因此我们需要先将远程的分支抓取过来
  11. git push 将本地分支上传到远程并合并 git push origin master 将本地分支上传到origin主机的master分
  12. gitHub: git config --global http.sslVerify "false" 在push时应对OpenSSL SSL_read: Connection was reset, errno 10054的一种方法
